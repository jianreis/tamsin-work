## The future-state journey, end to end (the spine)
The experience this vault serves design for, stage by stage. Each links to the
knowledge that constrains and inspires it.

`PRESCRIBE → SUBMIT → VALIDATE/CLINICAL CHECK → FUND/CLAIM → PAY → DISPENSE →
COLLECT or DELIVER → ADHERE → REFILL/RENEW`

| Stage | Tag | Key knowledge |
|---|---|---|
| Prescribe (script created) | `journey/prescribe` | e-prescribing, telehealth (regulatory) |
| Submit (upload/transfer) | `journey/submit` | Dis-Chem app, WhatsApp commerce, e-Rx law |
| Validate + clinical check | `journey/validate` | GPP, pharmacist oversight, scheduling |
| Fund / claim | — | medical schemes, PMB/CDL, real-time claims |
| Pay | `journey/pay` | SEP, dispensing fee, cash vs scheme |
| Dispense | `journey/dispense` | dispensing best practice, automation |
| Collect / Deliver | `journey/collect` `journey/deliver` | PackMyMeds, DeliverD, last-mile in SA |
| Adhere | `journey/adhere` | adherence packaging & behavioural design |
| Refill / Renew | `journey/refill` | repeat dispensing rules, predictive refills |
