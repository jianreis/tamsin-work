# Summary

The authoritative accessibility standard (**WCAG 2.2**, POUR principles) and
W3C/WAI's guidance on **older users** — whose needs (vision, contrast, dexterity,
cognition) overlap strongly with the elderly chronic-medication population a SA
pharmacy serves. Defines the concrete, testable criteria (contrast ratios, touch
targets, text resizing) for inclusive interface design.

# Key claims
- **WCAG POUR principles:** content must be **Perceivable, Operable,
  Understandable, Robust**; success criteria are graded **A / AA / AAA**, with
  **AA the common baseline** (and the legal target for many public services).
- **Contrast:** WCAG AA requires a text contrast ratio of at least **4.5:1**
  (3:1 for large text); AAA is 7:1. Older eyes lose contrast sensitivity, so
  high contrast helps.
- **Text:** users must be able to **resize text up to 200%** without loss of
  function; recommended base size ≈ **16px+**; avoid low-contrast grey-on-grey.
- **Target size (WCAG 2.2):** interactive targets should be at least **24×24 CSS
  px** (AA), with ~44×44 px a common practical recommendation for touch.
- **Older-user design that helps everyone:** large readable text, clear/explicit
  link text, plain neutral backgrounds, **selection rather than free-text input**,
  step-by-step instructions, icon + text labels, generous spacing, forgiving
  forms. These improved older-user performance **without harming younger users**.
- **Understandable / cognitive:** plain language, consistent navigation,
  predictable behaviour, and clear error recovery reduce load for everyone,
  especially users with lower digital literacy.
